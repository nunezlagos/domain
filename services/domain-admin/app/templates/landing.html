El problema

¿Te suena familiar?

Cada herramienta tiene su propio archivo de reglas, sus propios prompts, sus propias instrucciones. Cuanto más crecés, más caos.

📄

Reglas dispersas en todos lados

.cursorrules, CLAUDE.md, system prompts, archivos AGENTS.md. Cada cliente, cada proyecto, cada repo tiene su propia versión de "las reglas". Nada está centralizado.

🔁

Skills duplicadas sin versionar

¿Cuántas veces reescribiste la misma skill de "send email" o "query database"? Sin un registro central, cada equipo la reimplementa y no hay forma de saber cuál versión funciona mejor.

🔍

Sin trazabilidad en producción

Tus agentes hacen cosas. ¿Cuántas llamadas? ¿Con qué tasa de éxito? ¿Cuándo falló esa skill la semana pasada? Sin observabilidad, estás operando a ciegas.

Solución

Todo lo que necesitás,
en un solo servidor

Domain MCP es el sistema nervioso de tu infraestructura de agentes. Gobernás, observás y evolucionás desde un panel unificado.

Un servidor MCP para gobernarlos a todos

Registrás Domain MCP una sola vez en cada cliente (Claude Code, Cursor, OpenCode…) y automáticamente tenés acceso a todas tus skills, políticas y flujos. Sin duplicar configuraciones, sin archivos sueltos por el repo.

Políticas de gobernanza

Definí reglas a nivel plataforma o por proyecto: no PII en logs, restricciones de seguridad, compliance. El MCP las inyecta en el contexto de cada agente automáticamente.

Flujos multi-fase

Pipelines SDD, issue intake, deploy workflows. Definís las fases, las herramientas MCP disponibles en cada una, y el agente las ejecuta de forma estructurada y trazable.

Observabilidad real

Métricas por skill (calls, success rate, latencia), feedback de usuarios, A/B testing automático entre versiones. Sabés exactamente qué funciona y qué no.

Crons inteligentes

Trabajos programados que ejecutan skills o flujos. Aggregación de feedback, inferencia de relaciones, evaluación de calidad con LLM-as-judge. Todo configurable y observable.

100% self-hosted

Tus datos en tu VPS. Postgres, MinIO, todo bajo tu control. Sin vendor lock-in, sin métricas enviadas a terceros. Instalás en un Ubuntu y es tuyo para siempre.

Instalación

Dos comandos.
Stack completo.

Primero levantás el servidor en tu VPS. Después configurás tu laptop para que los clientes MCP apunten a él.

1

Instalá el stack completo en tu VPS

Un solo comando instala Postgres, MinIO, el servidor MCP y el panel de administración. Genera credenciales automáticamente y levanta todos los servicios con systemd.

bash 
# Ejecutá en tu VPS con Ubuntu 22.04+
sudo bash <(curl -fsSL https://raw.githubusercontent.com/nunezlagos/domain/main/services/install.sh)
Idempotente: podés re-ejecutar para actualizar sin perder datos. Las credenciales generadas en la primera ejecución se preservan en /opt/services/.env.
2

Anotá tu IP y las credenciales

Al terminar, el instalador imprime la IP del servidor, el URL del panel y las credenciales generadas. Guardálas en tu password manager.

output 
# Al terminar verás algo así:
  Dashboard:  http://TU_IP/
  MCP HTTP:   http://TU_IP/mcp
  API:        http://TU_IP/api/v1/
1

Cloná el repo

El instalador de cliente vive en el mismo repo. Lo clonás una vez.

bash 
git clone https://github.com/nunezlagos/domain -b main
cd domain
2

Ejecutá el bootstrap con la URL de tu VPS

Detecta Go (o lo instala localmente sin sudo), compila el binario, y registra el MCP en todos los clientes que encuentre instalados.

bash 
./install-user/bootstrap.sh \
  --url http://TU_IP \
  --email tu@email.com
Registra automáticamente Claude Code, OpenCode, Cursor, Claude Desktop y más si están instalados. Idempotente: re-ejecutar actualiza sin duplicar.
